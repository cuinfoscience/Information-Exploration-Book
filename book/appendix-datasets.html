Appendix A — Datasets

A.1 What This Appendix Covers

This appendix is the reference you come back to whenever you need to remember how a dataset is structured or what its known quirks are. For every dataset used in the book, the entry below lists:

  • Source (publishing agency) and URL
  • Filename in the course repository
  • Chapters where the dataset is used
  • Structure — what each row represents and the full column list with types
  • Known quirks — the gotchas you need to handle before analysis

The “Known quirks” block for each dataset is your per-dataset troubleshooting guide. Read it before loading the file for the first time; return to it whenever a new error surprises you.

All datasets are available in the accompanying GitHub repository.

A.2 Colorado Cannabis Sales by County

Source: Colorado Department of Revenue, Marijuana Enforcement Division
URL: https://cdor.colorado.gov/data-and-reports/marijuana-data/marijuana-sales-reports
File: colorado_cannabis_sales_by_county.csv
Used in: Chapter 5, Chapter 9

This file contains monthly medical and retail cannabis sales totals for each Colorado county from 2014 to 2024. The original file is an Excel export with a two-row title block at the top and an eight-row footnote section at the bottom.

Structure: Each row represents one county’s sales for one month.

Column Type Description
Month integer Calendar month (1–12)
Year integer Calendar year (2014–2024)
County text Colorado county name
Medical Sales currency string Medical cannabis sales in dollars
Retail Sales currency string Retail cannabis sales in dollars

Known quirks:

  • Header row is on row 2 (zero-indexed), not row 0. Use header=2 in read_csv.
  • Eight footer rows contain footnotes and source information. Use skipfooter=8.
  • Sales columns are formatted as currency strings ($1,234,567) and must be stripped of $ and , before numeric conversion.
  • NR (Not Reported) appears in sales columns for counties with fewer than three licensees. Treat as missing with na_values=['NR'].
  • Rows with County == 'Total' are monthly statewide aggregates and should be removed before county-level analysis.
  • County names may contain leading or trailing whitespace.

A.9 Non-Tabular Data Sources

The five chapters in Part IV use specialized datasets — shapefiles, edge lists, text corpora, time series, and API responses. Each chapter’s examples assume one or more of the sources listed below.

A.9.1 Census TIGER/Line Shapefiles (Chapter 13)

Source: US Census Bureau URL: https://www.census.gov/geographies/mapping-files/time-series/geo/tiger-line-file.html Files: tl_2023_08_county.shp (Colorado counties) and companion .shx, .dbf, .prj files Used in: Chapter 13

County and state boundary polygons for the entire United States, republished annually. The Colorado-specific file (08 = state FIPS) is a good default for the book’s spatial examples.

Known quirks:

  • A shapefile is actually several files (.shp, .shx, .dbf, .prj). Keep them together in the same folder.
  • Default CRS is EPSG:4269 (NAD83 latitude/longitude). Reproject to EPSG:3857 for overlays on web basemaps.
  • Each county has a GEOID / COUNTYFP that matches the countyfips column in the State Demography Office data.

A.9.2 ICON / KONECT / Network Repository Edge Lists (Chapter 14)

Sources: ICON (CU Boulder), KONECT (Koblenz), Network Repository Example files: soc-karate.edges (Zachary’s karate club), soc-dolphins.edges (dolphin social network) Used in: Chapter 14

Small, curated network datasets suitable for teaching. Zachary’s karate club (34 nodes, 78 edges) is the single most common beginner example.

Known quirks:

  • Edge-list files often start with comment lines prefixed by % (KONECT) or #. Use comment='%' in pd.read_csv to skip them.
  • Many datasets ship as whitespace-delimited .edges files, not .csv. Pass sep=r'\s+' or sep=' ' to pd.read_csv.
  • Node IDs are typically integers. Cast consistently before building a graph to avoid “node not in graph” errors.

A.9.3 Project Gutenberg and NLTK Corpora (Chapter 15)

Sources: Project Gutenberg, NLTK built-in corpora Example files: 2021-biden.txt (Project Gutenberg), inaugural corpus (NLTK built-in) Used in: Chapter 15

The NLTK inaugural corpus contains every US presidential inaugural address (1789–present) as plain text, accessible via nltk.corpus.inaugural. Project Gutenberg distributes public-domain literature in UTF-8 text.

Known quirks:

  • Project Gutenberg files include a boilerplate header and footer that must be stripped before analysis.
  • NLTK corpora require a one-time nltk.download('inaugural') (and similar for other corpora and resources like punkt, stopwords, wordnet, vader_lexicon).
  • Encoding is UTF-8 for recent files; older Gutenberg texts may arrive as Latin-1. Pass encoding= explicitly.

A.9.4 FRED / World Bank / IMF Time Series (Chapter 16)

Sources: FRED, World Bank Open Data, IMF Data Example files: UNRATE.csv (US unemployment rate, monthly), CPIAUCSL.csv (CPI, monthly), HOUST.csv (housing starts, monthly) Used in: Chapter 16

FRED publishes thousands of US macroeconomic series as downloadable CSVs with two columns: DATE and the series value. World Bank data arrives in wide format (one column per year) and requires melt to reshape into long.

Known quirks:

  • FRED files use DATE as an ISO date column. Parse with pd.to_datetime(df['DATE']) and set as the index.
  • Some series have “.” or blank rows for missing observations (e.g., during data-collection gaps). Treat these as NaN via na_values=['.', ''].
  • World Bank CSVs include 4 metadata rows at the top. Use skiprows=4.
  • IMF files sometimes require joining a separate country-code reference file before they are usable.

A.9.5 Public APIs for Structured Data (Chapter 17)

Sources: FRED API, Census Bureau API, OpenAlex API, Wikipedia/MediaWiki API, Open Library API Example file: example_openalex_response.json (cached API response), any static Wikipedia page for HTML scraping Used in: Chapter 17

All five APIs above are free; most require only a User-Agent header (or, for the OpenAlex “polite pool,” a mailto query parameter). The chapter includes worked examples against OpenAlex.

Known quirks:

  • FRED, Census, and Open Library require registration for an API key. OpenAlex and Wikipedia are unauthenticated.
  • Every API paginates differently — some use page=N, some use a cursor token, some use Link: headers. Read the docs before looping.
  • Rate limits are real. Respect them with time.sleep() between requests, and cache responses to disk during development.
  • API responses are JSON; flatten with pd.json_normalize() to get a clean DataFrame.
NotePublic Interest Context: Exemption

The datasets described in this appendix represent what is publicly available — what agencies have chosen to publish, in the formats they chose to publish it. That is a narrower set than what exists or what would be useful for public accountability.

Noticeably absent from what is publicly available at a useful granularity:

  • Housing displacement records at the parcel level — public records exist, but parcel-level linkage to eviction or code enforcement actions is rarely published.
  • Policing data at the incident level — Colorado requires some reporting, but stop-and-frisk data, use-of-force details, and officer-level records are inconsistently available across jurisdictions.
  • Detention and incarceration populations — county jail populations are not systematically published in machine-readable form.
  • Environmental health data — air quality monitoring stations cover urban centers but are sparse in rural counties and on Tribal lands.

Exemption — data that does not exist publicly, or is not released — is itself a data quality problem. The questions you cannot answer because the data is missing are often the most important ones to ask.

When you finish an analysis, it is worth asking: what would you study if you had access to data that is not currently available? That question is as analytically important as what you did study. See Chapter 4 for the broader framework.

- Multiple rows per parcel account number (one per building section).

### Residential Sales

**Source:** Boulder County Assessor's Office  
**URL:** <https://bouldercounty.gov/property-and-land/assessor/sales/>  
**File:** `ar-rec-2025-residential.xlsx`  
**Used in:** @sec-py-cleaning

Residential property sales records.

| Column | Type | Description |
|--------|------|-------------|
| ACCOUNTNO | text | Property account number (has trailing whitespace) |
| PROPERTY_ADDRESS | text | Street address |
| LOCCITY | text | City |
| SALE_DATE | datetime | Date of sale |
| SALE_PRICE | mixed | Sale price (contains `$`, commas, and text like `PRIVATE SALE`) |

**Known quirks:**

- `ACCOUNTNO` has trailing whitespace that must be stripped.
- `SALE_PRICE` contains currency formatting and non-numeric values like `PRIVATE SALE`.
- Multiple rows per `ACCOUNTNO` (representing different sale transactions over time).

## Non-Tabular Data Sources

The five chapters in Part IV use specialized datasets — shapefiles, edge lists, text corpora, time series, and API responses. Each chapter's examples assume one or more of the sources listed below.

### Census TIGER/Line Shapefiles (Chapter 13)

**Source:** US Census Bureau
**URL:** <https://www.census.gov/geographies/mapping-files/time-series/geo/tiger-line-file.html>
**Files:** `tl_2023_08_county.shp` (Colorado counties) and companion `.shx`, `.dbf`, `.prj` files
**Used in:** @sec-spatial

County and state boundary polygons for the entire United States, republished annually. The Colorado-specific file (`08` = state FIPS) is a good default for the book's spatial examples.

**Known quirks:**

- A shapefile is actually several files (`.shp`, `.shx`, `.dbf`, `.prj`). Keep them together in the same folder.
- Default CRS is `EPSG:4269` (NAD83 latitude/longitude). Reproject to `EPSG:3857` for overlays on web basemaps.
- Each county has a `GEOID` / `COUNTYFP` that matches the `countyfips` column in the State Demography Office data.

### ICON / KONECT / Network Repository Edge Lists (Chapter 14)

**Sources:** [ICON](https://icon.colorado.edu/) (CU Boulder), [KONECT](http://konect.cc/) (Koblenz), [Network Repository](https://networkrepository.com/)
**Example files:** `soc-karate.edges` (Zachary's karate club), `soc-dolphins.edges` (dolphin social network)
**Used in:** @sec-dyadic

Small, curated network datasets suitable for teaching. Zachary's karate club (34 nodes, 78 edges) is the single most common beginner example.

**Known quirks:**

- Edge-list files often start with comment lines prefixed by `%` (KONECT) or `#`. Use `comment='%'` in `pd.read_csv` to skip them.
- Many datasets ship as whitespace-delimited `.edges` files, not `.csv`. Pass `sep=r'\s+'` or `sep=' '` to `pd.read_csv`.
- Node IDs are typically integers. Cast consistently before building a graph to avoid "node not in graph" errors.

### Project Gutenberg and NLTK Corpora (Chapter 15)

**Sources:** [Project Gutenberg](https://www.gutenberg.org/), [NLTK built-in corpora](https://www.nltk.org/nltk_data/)
**Example files:** `2021-biden.txt` (Project Gutenberg), `inaugural` corpus (NLTK built-in)
**Used in:** @sec-textual

The NLTK `inaugural` corpus contains every US presidential inaugural address (1789–present) as plain text, accessible via `nltk.corpus.inaugural`. Project Gutenberg distributes public-domain literature in UTF-8 text.

**Known quirks:**

- Project Gutenberg files include a boilerplate header and footer that must be stripped before analysis.
- NLTK corpora require a one-time `nltk.download('inaugural')` (and similar for other corpora and resources like `punkt`, `stopwords`, `wordnet`, `vader_lexicon`).
- Encoding is UTF-8 for recent files; older Gutenberg texts may arrive as Latin-1. Pass `encoding=` explicitly.

### FRED / World Bank / IMF Time Series (Chapter 16)

**Sources:** [FRED](https://fred.stlouisfed.org/), [World Bank Open Data](https://data.worldbank.org/), [IMF Data](https://www.imf.org/en/Data)
**Example files:** `UNRATE.csv` (US unemployment rate, monthly), `CPIAUCSL.csv` (CPI, monthly), `HOUST.csv` (housing starts, monthly)
**Used in:** @sec-temporal

FRED publishes thousands of US macroeconomic series as downloadable CSVs with two columns: `DATE` and the series value. World Bank data arrives in wide format (one column per year) and requires `melt` to reshape into long.

**Known quirks:**

- FRED files use `DATE` as an ISO date column. Parse with `pd.to_datetime(df['DATE'])` and set as the index.
- Some series have "." or blank rows for missing observations (e.g., during data-collection gaps). Treat these as `NaN` via `na_values=['.', '']`.
- World Bank CSVs include 4 metadata rows at the top. Use `skiprows=4`.
- IMF files sometimes require joining a separate country-code reference file before they are usable.

### Public APIs for Structured Data (Chapter 17)

**Sources:** [FRED API](https://fred.stlouisfed.org/docs/api/fred/), [Census Bureau API](https://www.census.gov/data/developers/data-sets.html), [OpenAlex API](https://docs.openalex.org/), [Wikipedia/MediaWiki API](https://www.mediawiki.org/wiki/API:Main_page), [Open Library API](https://openlibrary.org/developers/api)
**Example file:** `example_openalex_response.json` (cached API response), any static Wikipedia page for HTML scraping
**Used in:** @sec-structured

All five APIs above are free; most require only a `User-Agent` header (or, for the OpenAlex "polite pool," a `mailto` query parameter). The chapter includes worked examples against OpenAlex.

**Known quirks:**

- FRED, Census, and Open Library require registration for an API key. OpenAlex and Wikipedia are unauthenticated.
- Every API paginates differently — some use `page=N`, some use a cursor token, some use `Link:` headers. Read the docs before looping.
- Rate limits are real. Respect them with `time.sleep()` between requests, and cache responses to disk during development.
- API responses are JSON; flatten with `pd.json_normalize()` to get a clean DataFrame.

::: {.callout-note}
## Public Interest Context: Exemption

The datasets described in this appendix represent what is publicly available — what agencies have chosen to publish, in the formats they chose to publish it. That is a narrower set than what exists or what would be useful for public accountability.

Noticeably absent from what is publicly available at a useful granularity:

- **Housing displacement records** at the parcel level — public records exist, but parcel-level linkage to eviction or code enforcement actions is rarely published.
- **Policing data at the incident level** — Colorado requires some reporting, but stop-and-frisk data, use-of-force details, and officer-level records are inconsistently available across jurisdictions.
- **Detention and incarceration populations** — county jail populations are not systematically published in machine-readable form.
- **Environmental health data** — air quality monitoring stations cover urban centers but are sparse in rural counties and on Tribal lands.

Exemption — data that does not exist publicly, or is not released — is itself a data quality problem. The questions you cannot answer because the data is missing are often the most important ones to ask.

When you finish an analysis, it is worth asking: what would you study if you had access to data that is not currently available? That question is as analytically important as what you did study. See @sec-public-interest for the broader framework.
:::
