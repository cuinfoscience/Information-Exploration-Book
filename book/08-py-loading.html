8  Loading Data with pandas

8.1 Learning Objectives

By the end of this chapter, you will be able to:

  • Load CSV and Excel files into a pandas DataFrame using pd.read_csv() and pd.read_excel().
  • Control how pandas interprets a file using header, skipfooter, na_values, sep, dtype, and encoding.
  • Run the post-load checklist — shape, headers, types, missingness, distributions, duplicates, plausibility, and a quick visual scan — on every new dataset.
  • Diagnose FileNotFoundError, DtypeWarning, and other loading errors, and choose an appropriate fix.
  • Navigate pandas’ documentation using help(), the ? operator, and the online API reference.

8.2 Why Loading Is Harder Than It Looks

Every analysis begins with loading data — getting information from a file on disk into a pandas DataFrame where you can inspect and manipulate it. This chapter covers the mechanics of loading CSV and Excel files using pd.read_csv() and pd.read_excel(), the parameters that control how pandas interprets the file, and the post-load inspection routine that catches problems before they propagate into your analysis.

Loading data is where most silent errors originate. A column that looks numeric but was loaded as text, a header row mistakenly treated as data, a delimiter that splits values in the wrong places — these problems are invisible unless you actively look for them. The discipline of inspecting data immediately after loading is one of the most valuable habits you will develop in this course.

This chapter builds on the project-organization and terminal skills from Chapter 2 and Chapter 1. If a pd.read_csv call fails with FileNotFoundError, review those chapters — the fix is almost always a path or working-directory issue.

8.4 How to Read pandas Documentation

Before you run your first load, take a minute to learn how to read the documentation. You will come back to the docs constantly — not because you are a beginner, but because experienced data scientists do the same. Nobody memorizes every parameter of pd.read_csv (there are more than 30).

There are three ways to access documentation from inside a notebook:

  1. The ? operator (IPython / Jupyter only): type pd.read_csv? in a cell and run it. A help panel appears with the function’s signature and docstring.
  2. The help() function (works anywhere): help(pd.read_csv) prints the same information as text.
  3. The online docs: pandas.pydata.org/docs/reference/api/pandas.read_csv.html shows the same content formatted for the web, plus examples and cross-references.

A function signature looks like this:

pd.read_csv(filepath_or_buffer, *, sep=NoDefault.no_default, header='infer',
            names=None, index_col=None, usecols=None, dtype=None,
            engine=None, ..., encoding=None, skipfooter=0, ...)

The first argument (filepath_or_buffer) is required — you must pass it. Everything after the * is a keyword argument — it has a default value and you can leave it out unless you need to change it. The documentation tells you what each parameter does, what types it accepts, and what the default behavior is.

TipDocs are a reference, not a tutorial

Pandas documentation is terse by design. It describes behavior, not motivation. When you are learning a new function, skim the signature and the first paragraph, then jump to the examples at the bottom — that is usually the fastest way to understand what the function does. Come back to the parameter list when you hit a specific problem.

8.6 The Post-Load Checklist

Before you touch any data, run through this checklist. It takes less than a minute and catches the majority of loading problems.

  • Shape: df.shape — does the row and column count match expectations?
  • Headers: df.columns.tolist() — are column names unique, non-empty, and meaningful?
  • Types: df.dtypes — are numbers numeric, dates datetime, and text object?
  • Missingness: df.isna().sum() — how many missing values per column?
  • Distributions: df['col'].value_counts() — do categorical values look right?
  • Duplicates: df.duplicated(subset=[key_cols]).sum() — is there a candidate key?
  • Plausibility: df.describe() — any impossible ranges, negative counts, future dates?
  • Quick look: df.head(), df.tail(), df.sample(5) — do rows look right?

You will apply this checklist in every loading chapter in this book, in both Python and SQL. Memorize it.

8.7 Fixing the Load with read_csv Parameters

The pd.read_csv() function has dozens of parameters that control how the file is interpreted. For most real-world files, you will need to specify at least a few of them. Here are the most important ones.

8.7.1 header: where the column names are

If the header row is not the first row of the file, specify its zero-indexed position:

df = pd.read_csv(
    'colorado_cannabis_sales_by_county.csv',
    header=2
)

df.head()

8.7.2 skipfooter: removing trailing rows

Government data files often have footnotes, totals, or metadata appended to the bottom. The skipfooter parameter removes a specified number of rows from the end. Note that skipfooter requires setting engine='python' because the default C parser does not support it.

df = pd.read_csv(
    'colorado_cannabis_sales_by_county.csv',
    header=2,
    skipfooter=8,
    engine='python'
)

df.tail()

8.7.3 na_values: declaring custom missing value codes

Many datasets use codes like NR (Not Reported), -999, ., or empty strings to represent missing values. The na_values parameter tells pandas to treat these as NaN:

df = pd.read_csv(
    'colorado_cannabis_sales_by_county.csv',
    header=2,
    skipfooter=8,
    engine='python',
    na_values=['NR']
)

df.tail()

8.7.4 sep: specifying the delimiter

CSV stands for “comma-separated values,” but many files use other delimiters — tabs (\t), pipes (|), or semicolons (;). If your file uses something other than a comma, specify it with sep:

# Example: pipe-delimited file
# df = pd.read_csv('voter_data.txt', sep='|')

8.7.5 dtype: controlling column types

By default, pandas guesses each column’s type. You can override this with a dictionary mapping column names to types. This is especially useful for columns that look numeric but should remain strings — like zip codes or ID numbers that have leading zeros:

# Example: preserve leading zeros in zip codes
# df = pd.read_csv('data.csv', dtype={'zip_code': str, 'voter_id': str})

8.8 Building Up Iteratively

Notice the pattern above: you load the file, check the result, add a parameter to fix a problem, and reload. This iterative approach — get something to work, then move on to the next issue — is how experienced analysts work with messy files.

In a finished notebook, you would consolidate everything into a single cell. But when you are first working with a new dataset, spread the steps across multiple cells so you can see the effect of each parameter.

8.9 What the Parameters Fixed — and What Remains

The read_csv parameters handle structural problems at the boundary between file and DataFrame: they put the header in the right place, skip rows that are not data, and mark sentinel values as missing. Run the checklist again to see what they accomplished:

NoteFrom loading to cleaning

The column names, currency formatting, and statewide aggregate rows still need attention. Those are post-load cleaning operations — renaming columns, stripping currency symbols, filtering rows — covered in Chapter 9, which begins by completing this dataset before moving to more complex cleaning scenarios.

8.10 Running the Post-Load Checklist

print('Shape:', df.shape)
print('\nDtypes:')
print(df.dtypes)   # object dtype for numeric columns means they were loaded as strings
print('\nMissing values:')
print(df.isna().sum())  # count NaN values per column; NR entries should now show as NaN
# value_counts() for categorical columns — check that values look sensible
# (12 unique months, expected years, no garbled county names)
for col in ['Month', 'Year', 'County']:
    print(df[col].value_counts(), '\n\n')
# describe() for numeric-looking columns — note: these are still strings at this point
# so describe() shows string statistics (count, unique, top, freq) not min/max
for col in ['Medical', 'Retail']:
    print(df[col].describe(), '\n\n')

8.11 A Quick Preview: What You Can Do with Clean Data

With the data loaded and cleaned, you can immediately begin exploring. Here is a pivot table and a simple visualization — both of which we will cover in depth in later chapters:

# pivot_table() aggregates and reshapes in one step:
# index='County' → one row per county
# columns='Year' → one column per year
# values='Retail' → the cell values are retail sales
# aggfunc='sum' → sum all months within each county-year cell
pivot_df = pd.pivot_table(
    data=df,
    index='County',
    columns='Year',
    values='Retail',
    aggfunc='sum'
)

pivot_df.head()
# .loc['Las Animas'] selects the row for Las Animas County
# .plot() with a datetime-like index creates a line chart by default
pivot_df.loc['Las Animas'].plot(title='Las Animas County Retail Cannabis Sales by Year')

8.13 Common Issues in This Chapter

WarningFileNotFoundError: [Errno 2] No such file or directory

What it means: Pandas looked for the file at the path you gave and did not find it.

How to fix it: Run import os; print(os.getcwd()) to see where your notebook is looking. Confirm the file name spelling and extension. Use a relative path like '../data/raw/filename.csv' if the file lives in a sibling directory. See Chapter 5.

WarningDtypeWarning: Columns (N) have mixed types

What it means: Pandas found both numeric and non-numeric values in the same column and fell back to object (string) type. Your later arithmetic will silently fail.

How to fix it: Inspect the column with df['col'].value_counts(). If the mixing comes from a stray header row or footer, set skiprows or skipfooter. If the column has sentinel codes like NR or -999, pass them via na_values. If you want to force the type, use dtype={'col': str} (or float, int, etc.).

WarningUnicodeDecodeError: 'utf-8' codec can't decode byte

What it means: The file uses a non-UTF-8 encoding (common in files exported from older Windows software or government systems).

How to fix it: Try encoding='latin-1' or encoding='cp1252' in your pd.read_csv call. If you are on the command line, file my_data.csv (macOS/Linux) reports the encoding. Re-save the file as UTF-8 if you will use it repeatedly.

WarningNumbers import as strings because of $ or ,

What it means: A column like $1,234,567 is stored as text. df['Retail'] + 100 will concatenate a string rather than add a number.

How to fix it: Strip the currency/comma characters, then convert: df['Retail'] = pd.to_numeric(df['Retail'].str.replace(r'[$,]', '', regex=True), errors='coerce'). See Chapter 9 for the full pattern.

8.14 Key Takeaways

  • pd.read_csv and pd.read_excel have dozens of parameters; you rarely need all of them, but knowing header, skipfooter, na_values, sep, dtype, and encoding will cover most real-world files.
  • Run the post-load checklist on every new dataset: shape, headers, types, missingness, distributions, duplicates, plausibility, head/tail/sample.
  • Load → inspect → adjust → reload. This iterative pattern is how experienced analysts work with messy files.
  • Loading errors are the easiest class of errors to fix because the diagnostic message is usually precise. When in doubt, read the error, check the docs, and consult Chapter 5.
  • Documentation is a reference, not a tutorial. Get comfortable with help(), ?, and the online API page — they will save you hours over the course.

```{python}
df.tail()
```

### `sep`: specifying the delimiter

CSV stands for "comma-separated values," but many files use other delimiters — tabs (`\t`), pipes (`|`), or semicolons (`;`). If your file uses something other than a comma, specify it with `sep`:

```{python}

```

### `dtype`: controlling column types

By default, pandas guesses each column's type. You can override this with a dictionary mapping column names to types. This is especially useful for columns that look numeric but should remain strings — like zip codes or ID numbers that have leading zeros:

```{python}

# df = pd.read_csv('data.csv', dtype={'zip_code': str, 'voter_id': str})
```

## The `read_csv` Parameters You Should Know

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `header` | Which row contains column names | `header=2` |
| `skiprows` | Rows to skip at the top | `skiprows=3` |
| `skipfooter` | Rows to skip at the bottom | `skipfooter=8` (requires `engine='python'`) |
| `sep` | Column delimiter | `sep='|'`, `sep='\t'` |
| `na_values` | Custom missing value codes | `na_values=['NR', '-999', '.']` |
| `dtype` | Force column types | `dtype={'zip': str}` |
| `encoding` | Character encoding | `encoding='latin-1'` |
| `usecols` | Load only specific columns | `usecols=['County', 'Year', 'Sales']` |

## Common Issues in This Chapter

::: {.callout-warning}
## `FileNotFoundError: [Errno 2] No such file or directory`
**What it means:** Pandas looked for the file at the path you gave and did not find it.

**How to fix it:** Run `import os; print(os.getcwd())` to see where your notebook is looking. Confirm the file name spelling and extension. Use a relative path like `'../data/raw/filename.csv'` if the file lives in a sibling directory. See @sec-errors.
:::

::: {.callout-warning}
## `DtypeWarning: Columns (N) have mixed types`
**What it means:** Pandas found both numeric and non-numeric values in the same column and fell back to `object` (string) type. Your later arithmetic will silently fail.

**How to fix it:** Inspect the column with `df['col'].value_counts()`. If the mixing comes from a stray header row or footer, set `skiprows` or `skipfooter`. If the column has sentinel codes like `NR` or `-999`, pass them via `na_values`. If you want to force the type, use `dtype={'col': str}` (or `float`, `int`, etc.).
:::

::: {.callout-warning}
## `UnicodeDecodeError: 'utf-8' codec can't decode byte`
**What it means:** The file uses a non-UTF-8 encoding (common in files exported from older Windows software or government systems).

**How to fix it:** Try `encoding='latin-1'` or `encoding='cp1252'` in your `pd.read_csv` call. If you are on the command line, `file my_data.csv` (macOS/Linux) reports the encoding. Re-save the file as UTF-8 if you will use it repeatedly.
:::

::: {.callout-warning}
## Numbers import as strings because of `$` or `,`
**What it means:** A column like `$1,234,567` is stored as text. `df['Retail'] + 100` will concatenate a string rather than add a number.

**How to fix it:** Strip the currency/comma characters, then convert: `df['Retail'] = pd.to_numeric(df['Retail'].str.replace(r'[$,]', '', regex=True), errors='coerce')`. See @sec-py-cleaning for the full pattern.
:::

## Key Takeaways

- `pd.read_csv` and `pd.read_excel` have dozens of parameters; you rarely need all of them, but knowing `header`, `skipfooter`, `na_values`, `sep`, `dtype`, and `encoding` will cover most real-world files.
- Run the **post-load checklist** on every new dataset: shape, headers, types, missingness, distributions, duplicates, plausibility, head/tail/sample.
- Load → inspect → adjust → reload. This iterative pattern is how experienced analysts work with messy files.
- Loading errors are the easiest class of errors to fix because the diagnostic message is usually precise. When in doubt, read the error, check the docs, and consult @sec-errors.
- Documentation is a reference, not a tutorial. Get comfortable with `help()`, `?`, and the online API page — they will save you hours over the course.
