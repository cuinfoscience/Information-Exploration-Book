Appendix B — Where to Go Next

You have reached the end of this book. You can load messy CSVs, clean them systematically, derive new metrics, merge multi-source tables with an audit trail, and translate between pandas and SQL. You can run principled statistical comparisons and build plots that communicate what you found. That is a real toolkit — and it opens the door to several adjacent families of data work that build directly on these skills.

This appendix is a short, curated guide to the libraries you would learn next, organized by the kind of data they help you work with. Each section gives you a one-paragraph orientation, the official documentation link, and one beginner-friendly tutorial to start with. None of these libraries are covered in this book; all of them reward the habits — inspect before acting, validate after every transform, document your decisions — that you built here.

B.1 Learning Objectives

By the end of this appendix, you will be able to:

  • Name the standard Python library for each of spatial, network, web-scraping, text, time-series, and regression work.
  • Locate each library’s official documentation and a beginner tutorial in seconds when you need it.
  • Recognize which library is the right next step for a new analytical question you encounter.

B.2 A Note Before You Dive In

Each of these libraries is a full toolkit in its own right — comparable in depth to pandas. You do not need to learn all of them. Pick the one that matches a project you actually want to do, follow its own tutorial, and come back here when a different question calls for a different tool. Depth of use beats breadth of exposure every time.

If the library you need is not listed here, the next step is almost always: search for <data type> python library on your preferred search engine, pick the library with the most recent releases and a clear user guide, and skim its getting-started page. You have already built the skills to evaluate a new library by its documentation quality.

B.3 Spatial Data — GeoPandas

When your data has locations attached — addresses, coordinates, county polygons, census tracts — you are working with spatial data. GeoPandas extends pandas with a geometry column so that every row can carry a shape (a point, a line, a polygon) alongside its attribute data. You can compute distances, check whether a point falls inside a polygon, merge tables on spatial proximity, and produce maps directly from your DataFrame.

GeoPandas feels immediately familiar if you know pandas — same filtering, same grouping, same .merge() — plus a few new methods like .sjoin() for spatial joins and .plot() for maps. Under the hood it uses Shapely for geometry operations and pyproj for coordinate reference systems.

  • Official docs: geopandas.org/en/stable/docs.html
  • Beginner tutorial: GeoPandas: Getting Started — an hour of your time and you will have produced a map from a shapefile.
  • Complementary: Folium for interactive web maps, and contextily for basemap tiles.

B.4 Network and Graph Data — NetworkX

When your data describes relationships between entities — emails between people, flights between airports, citations between papers — you are working with network data. NetworkX represents a network as a graph of nodes and edges, each of which can carry arbitrary attribute data. It supports the standard network measures (degree, centrality, clustering, shortest paths) and integrates cleanly with pandas and matplotlib.

The mental model is different from pandas: a graph is not a table. But you can always turn a graph into a table of edges or nodes, and back again. Start there before learning the more advanced algorithms.

  • Official docs: networkx.org/documentation/stable/
  • Beginner tutorial: NetworkX Tutorial — walks through creating a graph, adding attributes, and computing basic statistics.
  • Complementary: pyvis for interactive network visualizations and graph-tool for performance-critical work.

B.5 Web Scraping — requests + Beautiful Soup

When the data you want lives on a web page and the publisher does not offer a downloadable file or an API, you scrape it. requests handles the HTTP layer: fetching a URL, sending form data, managing authentication. Beautiful Soup parses the returned HTML into a tree you can navigate by tag, class, or CSS selector.

Scraping comes with its own set of responsibilities. Respect the site’s robots.txt and terms of service. Rate-limit your requests. Cache responses so you are not re-fetching the same page ten times during development. Treat scraped data with extra suspicion — the HTML structure changes more often than you want, and your parser will break.

  • requests docs: requests.readthedocs.io/en/latest/
  • Beautiful Soup docs: www.crummy.com/software/BeautifulSoup/bs4/doc/
  • Beginner tutorial: Real Python: Beautiful Soup — Web Scraper With Python is a complete walkthrough from a URL to a cleaned DataFrame.
  • When Beautiful Soup is not enough: Playwright for Python handles JavaScript-rendered pages. Scrapy is the industrial-strength framework for larger scraping projects.

B.6 Unstructured Text — NLTK and spaCy

When your data consists of documents, comments, transcripts, or any other free text, you need natural language processing (NLP) tools. NLTK (Natural Language Toolkit) is the classic introductory library: tokenization, stemming, stopword removal, frequency analysis, and a rich collection of corpora and lexicons. It is slower than modern alternatives but unbeatable for learning the concepts.

spaCy is the production-oriented successor: fast, accurate, and designed around pretrained statistical models that handle part-of-speech tagging, named-entity recognition, and dependency parsing out of the box. For most real text work today, spaCy is the better starting point.

  • NLTK docs: www.nltk.org/ and the free NLTK Book.
  • spaCy docs: spacy.io/usage.
  • Beginner tutorial: Real Python: Natural Language Processing With Python’s NLTK Package.
  • Further: For modern embedding-based NLP (sentence similarity, classification, clustering), look at Hugging Face Transformers and sentence-transformers.

B.7 Time-Series Forecasting — Prophet

When your data is a time series and you need to forecast future values — sales next quarter, web traffic next month, inventory demand next week — Prophet (originally from Facebook, now maintained by Meta) is a reasonable default. It handles seasonality, holidays, and trend changes automatically, and exposes a simple fit / predict API that fits in naturally with pandas.

Prophet is not the only option. For classical statistical models, statsmodels has ARIMA, seasonal decomposition, and state-space models. For machine-learning approaches, sktime provides a unified interface to many forecasting algorithms. Start with Prophet to get something working; move to the others when you need specific behavior Prophet does not provide.

  • Official docs: facebook.github.io/prophet/docs/quick_start.html
  • Beginner tutorial: Prophet Quick Start — fit a model, make a forecast, inspect the trend and seasonality in one page.
  • Complementary: statsmodels time-series analysis and sktime.

B.8 Extended Statistical Modeling — statsmodels

scipy.stats covers the tests introduced in Chapter 18, but it stops short of what you will eventually need: multiple linear regression, logistic regression, generalized linear models, ANOVA with interactions, fixed and random effects, and time-series models. statsmodels is where those tools live in Python.

The interface has a strong R-like flavor: you specify models with a formula ('y ~ x1 + x2 + x1:x2'), fit them, and inspect the results with a summary() that lists every coefficient, standard error, and confidence interval. If you have ever used R or Stata for regression, statsmodels will feel familiar.

  • Official docs: statsmodels.org/stable/
  • Beginner tutorial: statsmodels User’s Guide.
  • Complementary: scikit-learn for machine-learning-oriented regression and classification (cross-validation, pipelines, model selection) — a natural next stop once your questions shift from inference to prediction.

B.9 Machine Learning — scikit-learn

If statsmodels is for inference (what coefficients explain my outcome?), scikit-learn is for prediction (given these inputs, what outcome should I expect?). It ships with almost every classical ML algorithm — linear and logistic regression, decision trees, random forests, gradient boosting, clustering, dimensionality reduction — behind a consistent fit / predict / score API.

For most tabular ML problems in 2026, the workflow is: clean and feature-engineer in pandas, pipeline transformations with ColumnTransformer, fit a model (often HistGradientBoostingRegressor or RandomForestClassifier), evaluate with cross-validation, and inspect feature importances. scikit-learn covers all of this.

  • Official docs: scikit-learn.org/stable/
  • Beginner tutorial: Getting Started with scikit-learn.
  • Complementary: XGBoost and LightGBM for gradient boosting at scale.

B.10 A Final Word

Every library on this list is someone’s years of careful work, made free for you to use. The skill that makes you effective with all of them is not memorization — it is the discipline of reading documentation, validating outputs, and writing code that you (or a collaborator) can understand six months from now. Those are exactly the habits this book was designed to build.

The next time you load a dataset that makes you say “I do not know where to start,” start the way every chapter in this book started: read the file, print its shape, look at the first few rows, check the dtypes, and ask what each row represents. The tools change; the rhythm does not.
