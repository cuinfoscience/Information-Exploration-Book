4  Python Environments

4.1 Learning Objectives

By the end of this chapter, you will be able to:

  • Explain what a Python environment is and why it matters for reproducibility.
  • Use conda to list, install, and update packages in your active environment.
  • Distinguish between conda and pip and decide which to use when.
  • Diagnose ImportError / ModuleNotFoundError by comparing your notebook’s kernel to the environment where the package is installed.
  • Verify a working SQLite installation on macOS, Windows, and Linux.

4.2 What an Environment Is

When you installed Python — probably through Anaconda — you installed the Python interpreter along with hundreds of packages that come bundled with it. Pandas, NumPy, Matplotlib, Jupyter, and many others were all placed into a specific directory on your computer. This collection of a Python interpreter and its associated packages is called an environment.

Understanding environments matters because the tools in this book depend on specific packages being installed and accessible. When something does not work, the problem is often that the wrong environment is active, that a package is missing, or that two packages have conflicting version requirements. This chapter gives you enough knowledge to diagnose and fix these problems.

4.3 Anaconda and Conda

Anaconda is a distribution of Python designed for data science. It includes the conda package manager, which handles installing, updating, and removing packages. You have been using conda implicitly every time you open Anaconda Navigator or Anaconda Prompt.

To check which version of conda you have, open a terminal and run:

conda --version

To see which packages are installed in your current environment:

conda list

To install a new package:

conda install pandas

To update a package:

conda update pandas

4.4 The Base Environment

When you install Anaconda, it creates a default environment called base. This is the environment that activates when you open Anaconda Prompt. For the purposes of this book, the base environment contains everything you need.

You can verify which environment is active by looking at your terminal prompt — it typically shows the environment name in parentheses:

(base) $

If you see a different name, or no parentheses at all, you may be in a different environment or conda may not be activated.

4.5 Pip: The Other Package Manager

Python has a second package manager called pip. While conda manages both Python packages and non-Python dependencies (like C libraries), pip installs only Python packages from the Python Package Index (PyPI). In an Anaconda environment, prefer conda install when possible, and fall back to pip install only for packages that conda does not carry.

pip install some-package
Warning

Mixing conda and pip in the same environment can occasionally cause conflicts. If you must use pip, do so after installing everything you can with conda.

4.6 Checking What Is Installed

When a notebook gives you an ImportError or ModuleNotFoundError, it means the package is not installed in your current environment. To check:

conda list | grep pandas    # Mac/Linux
conda list | findstr pandas  # Windows

If the package is missing, install it with conda install package-name.

4.7 Jupyter and Kernels

When you open a Jupyter notebook, it runs code using a kernel — a specific Python environment. If you have multiple environments, each one can have its own kernel. A common source of confusion is having a package installed in one environment but running your notebook with a kernel from a different environment.

This is a surprisingly common problem. You install pandas with conda install pandas in your base environment, but Jupyter is running with a different kernel — and the install is invisible to it. The result is a ModuleNotFoundError even though you are confident pandas is installed somewhere.

To check which Python your notebook is using, run this in a code cell:

import sys
print(sys.executable)   # full path to the Python interpreter this kernel uses
print(sys.version)      # Python version number

The output path should point to the Python interpreter inside your intended environment. If it points to a path you do not recognize, or a different environment than you expected, switch the kernel using the Kernel menu (Jupyter Notebook) or the kernel picker in the top-right corner (VS Code or JupyterLab).

4.7.1 Verifying the active environment matches the kernel

A more complete check compares the environment name in your terminal prompt with the kernel path reported by sys.executable:

import sys
import pandas   # will fail with ImportError if pandas is not in this kernel's env

print('Python location:', sys.executable)
print('pandas version:', pandas.__version__)

If this cell runs without error, the kernel has access to pandas and is in a working state. If it fails, install the package in the environment the kernel is pointing to, not just any environment.

4.8 SQLite Installation

SQLite is used in the SQL chapters of this book. On macOS and Linux, SQLite is typically preinstalled. To check:

sqlite3 --version

On Windows, SQLite may not be installed by default. You can install it through conda:

conda install -c anaconda sqlite

Alternatively, download the precompiled binaries from sqlite.org/download.html, unzip the archive, and add the folder to your system PATH. After editing the PATH, open a new terminal window — the old window will not see the change.

4.9 Common Issues in This Chapter

Warningconda: command not found (or 'conda' is not recognized)

What it means: Your terminal cannot find the conda executable. On Windows, you are probably in Command Prompt or PowerShell instead of Anaconda Prompt. On macOS/Linux, conda is not on your PATH.

How to fix it: On Windows, open Anaconda Prompt from the Start menu. On macOS/Linux, run the one-time conda init command (see the conda docs) and restart your terminal.

WarningModuleNotFoundError: No module named 'pandas'

What it means: The package is not installed in the environment your notebook is using — which may not be the environment you installed it in.

How to fix it: Run import sys; print(sys.executable) in a notebook cell. Copy that path. Then in your terminal, confirm that path matches the environment where you installed pandas. If they differ, either install pandas in the notebook’s environment or switch the notebook’s kernel to the environment that has pandas.

WarningA package installed fine but still fails to import

What it means: The running Jupyter kernel was started before the install finished, so it is still pointing at the old environment state.

How to fix it: Restart the Jupyter kernel (Kernel → Restart) and re-run your imports. This is one of the most common “ghost” errors and is almost always solved by a kernel restart.

WarningConda is very slow to resolve an install

What it means: Conda is trying to solve a complex dependency graph. For large data-science environments, this can take minutes.

How to fix it: Wait. If it still hangs after several minutes, consider using mamba — a faster drop-in replacement for conda — or install a newer version of conda that uses the libmamba solver by default.

4.10 Key Takeaways

  • An environment is a specific Python interpreter plus a set of installed packages. Your notebook is always running in some environment; when things break, identifying which one is the first question.
  • Prefer conda install for scientific Python packages; fall back to pip install when conda does not carry a package.
  • ImportError / ModuleNotFoundError almost always means “wrong environment” or “kernel not restarted,” not “package broken.”
  • sys.executable inside a notebook is the definitive answer to “which Python am I actually running?”

4.11 Further Reading

Tip📚 Going deeper on Python environments
  • Conda user guide — docs.conda.io/projects/conda/en/stable/user-guide/ is the definitive reference. Start with “Getting started with conda” and “Managing environments.”
  • Pip user guide — pip.pypa.io/en/stable/user_guide/ documents pip for the cases where you need it.
  • Mamba — mamba.readthedocs.io is a faster, compatible alternative to conda for complex installs.
  • Real Python: Setting Up Python for Machine Learning on Windows — realpython.com/python-windows-machine-learning-setup/ is an especially useful walkthrough for Windows users.
  • Jupyter kernels — jupyter.readthedocs.io/en/latest/install/kernels.html explains how to register additional kernels for other environments.

NoteExercise 2: Install SQLite

Follow the instructions above to verify that SQLite is installed on your system. Run sqlite3 --version in a terminal. If it is not found, install it using conda or by downloading the binaries. Create a test database by running sqlite3 test.db ".tables" and verify that the file test.db appears in your directory.

import sys
print('Python executable:', sys.executable)
print('Python version:', sys.version)

import pandas
print('pandas version:', pandas.__version__)
  1. What path does sys.executable report? Does it contain the name of your conda environment (e.g., base)?
  2. Open a terminal and run conda activate base (or your environment name), then conda list | grep pandas. Does the version match what the notebook reported?
  3. If you can switch the notebook to a different kernel (from the Kernel menu or kernel picker), do so and re-run the cell. Did the path and version change?
